# 项目描述：星池指挥官与个人算力网

星池指挥官的目标，是把一台台分散的个人电脑接入同一个“个人算力网”。用户
不需要记住文件在哪台机器、哪一步做到哪里、哪个工具该怎么配置；用户只需要
说明目标，系统负责连接设备、拆分任务、维护进度，并把结果带回同一个工作台。

这个项目面向非专业用户。它应该让用户感觉是在使用一个连续的个人工作网络，
而不是在维护一组开发工具。

## 产品定位

从 `/root/pr_review/personal-compute-network.html` 的表达看，星池指挥官要回答的
不是“怎样自动安装 VS Code 和 Codex”，而是：

> 如何让办公室电脑、家里电脑、实验室机器和服务器变成同一个可持续工作的现场。

因此项目的核心价值是：

- **一个入口**：桌面快捷方式、右键菜单、浏览器控制台和 Linux CLI 都进入同一个工作体系。
- **多台设备**：每台设备都可以贡献自己的文件、环境、算力和在线时间。
- **自动拆任务**：复杂目标被拆成查资料、读文件、写初稿、检查遗漏、整理结果等小步骤。
- **持续推进**：早上开始的任务，晚上回家或换到另一台机器后仍能接着做。
- **结果回到一处**：用户看到的是讲义、汇报、清单、代码改动或反馈，而不是分散的中间文件。
- **用户可控**：登录、授权、在线状态、模型额度、更新和本地 slave 状态都应可见、可恢复。

## 本仓库的职责

本仓库是个人算力网的本地入口和安装交付层，主要负责：

1. 把用户机器接入 `agent.cs.ac.cn` 工作区。
2. 把 Codex / Codex Desktop 配置到 `code.ai.cs.ac.cn` 模型服务。
3. 安装、配置和启动本机可用的前端体验。
4. 启动并维护本地 token refresher / model proxy，避免长进程持有过期 access token。
5. 安装 Loom driver，并管理本机 slave，让本机文件夹可以作为可调度的工作现场。
6. 提供 Windows 常驻控制台、托盘、右键菜单和 Linux headless CLI。

## Windows 用户体验

Windows 安装包面向普通用户，安装后桌面快捷方式显示为“星池指挥官”。

首次启动时，星池指挥官会引导用户完成两类连接：

- **大模型连接**：通过 `code.cs.ac.cn` / `codeapi.cs.ac.cn` 完成 ModelServer 登录或授权。
- **星池工作区连接**：通过 `agent.cs.ac.cn` 完成 AgentServer 工作区登录。

完成后，应用会自动写入 Codex 配置，使模型请求通过本地代理访问：

```toml
[model_providers.modelserver]
name = "modelserver"
base_url = "http://127.0.0.1:53452/v1"
experimental_bearer_token = "<local proxy token>"
wire_api = "responses"
```

本地代理再读取当前有效的 ModelServer access token，并按模型把请求转发到 ModelServer
网关 `https://code.ai.cs.ac.cn/v1` 上对应的端点：

| 模型 | 上游端点 | 转换 |
|---|---|---|
| `gpt-5.5` | `/v1/responses` | 直通 |
| `deepseek-v4-pro` | `/v1/chat/completions` | Responses ⇄ Chat Completions |
| `glm-5.2` | `/v1/messages` | Responses ⇄ Anthropic Messages |

Codex 客户端始终只说 OpenAI Responses 协议，第三方模型不需要单独配置或额外网关。
默认模型可在「星池指挥官」控制台的模型选择卡切换，或用
`agentctl set-model <name>` / `agentserver set-model <name>` 直接改 `~/.codex/config.toml`，
新建的 Codex 对话立即生效（旧对话沿用创建时锁定的模型）。

代理同时维护 access token，让 Codex Desktop、Codex CLI 或极简 VS Code 中的长时间会话
不会因为 token 刷新而重启。

### 前端模式

当前 Windows 主线支持两类前端：

- **Codex Desktop 模式**：默认体验。安装并配置 Codex Desktop，完成后从星池指挥官打开。
- **极简 VS Code 模式**：为需要文件夹/终端式体验的用户准备。安装 VS Code、配置 Codex CLI、
  语言包和简化界面，只保留用户完成任务所需的最小视图。

右键文件夹“用星池指挥官打开”会根据当前模式打开对应前端，并确保本地控制台和
token refresher 已启动。

## Linux Headless 用户体验

Linux 版本面向无桌面服务器或远程开发机。用户解压后得到 `agentserver` 命令：

- 在某个目录执行 `agentserver`：把当前目录作为 foreground slave 启动。
- 执行 `agentserver install-driver`：把 driver MCP 写入当前用户的 `~/.codex/config.toml`。
- 执行 `agentserver switch-workspace`：重新选择远端 AgentServer 工作区。

Linux 首次使用时会通过 device code 和二维码完成登录：

- `codeapi.cs.ac.cn` 用于 ModelServer 侧 device login。
- `agent.cs.ac.cn` 用于 driver/slave 加入工作区。

如果环境变量 `OPENAI_API_KEY` 已经提供长期可用的模型 API key，Linux 命令优先使用直连
ModelServer 配置；否则启动单实例本地 proxy/refresher，并使用 `0600` 权限的 per-user
proxy token。

## 背后系统分工

- **星池指挥官 app**：安装、登录、配置、更新、本地控制台、前端启动、本机 slave 管理。
- **AgentServer**：保存工作区、设备状态、远程身份、driver/slave 注册和跨入口连续体验。
- **ModelServer**：提供模型账号、模型调用、项目额度、token 发放和用量边界。
- **Loom**：负责 driver/slave 协作，把用户目标拆成多步任务，并协调多个助手和多台机器执行。

用户不需要记住这些项目名。产品上只需要传达三件事：

1. 一个入口。
2. 一个连续工作台。
3. 复杂目标自动拆分并持续推进。

## 典型场景

### 教师备课

用户说：

> 帮我准备下周关于可再生能源的一节课，学生是高一，课堂时间 45 分钟。

系统可以把任务拆成：

- 从已有课件、网页资料和本地文档里筛选材料。
- 生成课堂导入、讲义提纲、案例材料、课堂提问和小测验。
- 检查内容是否过难、时间是否足够、还缺少哪些材料。
- 把可修改的课堂准备包带回同一个工作台。

### 办公汇报

用户说：

> 帮我准备周五项目例会的汇报，重点说明进展、风险和需要协调的事项。

系统可以把任务拆成：

- 读取会议纪要、项目表格、历史通知和指定参考文档。
- 提炼进展、问题、风险和下一步行动。
- 形成汇报初稿。
- 检查缺少数据、负责人不清、时间点不清楚的部分。

### 多地点连续工作

- 办公室电脑读取单位资料和表格。
- 家里电脑晚上继续改讲义或汇报。
- 实验室机器处理大文件或长时间任务。
- 浏览器控制台查看进度、补充要求、恢复失败步骤。

用户体验不是远程控制某台电脑，而是进入同一个工作现场。

## 设计原则

- **普通用户优先**：不要要求用户理解 OAuth、API key、MCP、driver/slave 或 token refresh。
- **复杂性进后台**：安装、登录、权限、更新、代理和进程管理由星池指挥官处理。
- **状态可见**：哪些电脑已接入、是否在线、正在做什么、是否需要重新登录，都要说清楚。
- **失败可恢复**：网络、授权、安装、更新或 slave 启动失败时，给出明确下一步。
- **不抢控制权**：用户可以暂停、恢复、删除本机 slave，也可以切换工作区或退出模型登录。

## 开发和测试

常用命令：

```bash
make build
make test
make cross-windows
make package
```

Windows 完整打包依赖 Wine + Inno Setup。测试 Windows 安装包时，应覆盖：

- 首次启动登录流程。
- Codex Desktop 默认模式。
- 极简 VS Code 模式。
- 右键“用星池指挥官打开”。
- 本地控制台、托盘、更新检查和 model proxy/refresher。
- 创建、启动、暂停、删除本机 slave。

Linux headless 测试应覆盖：

- 无 `OPENAI_API_KEY` 时的 device-code 登录和本地 proxy/refresher。
- 有 `OPENAI_API_KEY` 时的直连 ModelServer 配置。
- `agentserver` foreground slave。
- `agentserver install-driver` 和 `serve-driver-mcp`。
