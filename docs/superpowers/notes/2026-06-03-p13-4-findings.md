## Bugs Found (to fix in v1.0.1 / v2)

### Bug #1: Inno Setup installer returns 0xc0000409 in non-interactive session

Repro: SSH to Windows host, run `agentctl test-install-vscode` (which calls
`vscode.SilentInstall` → `Start-Process VSCodeUserSetup-x64-1.96.0.exe /VERYSILENT`).
Expected: exit 0, VS Code installed.
Actual: exit 0xc0000409 (STATUS_STACK_BUFFER_OVERRUN). VS Code IS installed
(verified via `code.cmd --version`), but the installer's main process crashes.

Cause: One of Inno Setup's spawned child processes (likely a UI hookup or
the post-install "runcode" stub) crashes in a non-interactive desktop session.
The install itself completes before the crash.

Fix for v1.0.1: After `vscode.SilentInstall` fails, re-run `vscode.Detect()`
— if VS Code is now present at the expected version, treat as success.

Location: `internal/ui/orchestrator_real.go:EnsureVSCode`

```go
if err := vscode.SilentInstall(ctx, cache, plan); err != nil {
    // Some Inno Setup configs return non-zero even on success; double-check.
    if det2, derr := vscode.Detect(); derr == nil && det2.Installed &&
        det2.Version == vscode.LockedVersion {
        // success path
    } else {
        return fmt.Errorf("install VS Code: %w", err)
    }
}
```

### Bug #2: `agentctl uninstall` deletes codex.exe in LocalAppDataRoot

Repro: Install → download codex → uninstall → reinstall → codex.exe gone,
must re-download 246MB.

Cause: `cmd/agentctl/cmd_uninstall.go` removes `p.LocalAppDataRoot` which
on Windows is `%LOCALAPPDATA%\agentserver-vscode\` — that's where the bin/
subdir holds codex.exe. Uninstall should remove our state, but probably
should preserve codex.exe (it's user-fetched data, not our state).

Fix options:
- (A) Move codex.exe to under `%LOCALAPPDATA%\Programs\agentserver-vscode\bin\`
  (inside the install dir, so install.ps1's uninstall already handles it,
  and re-install re-creates) — but then ConfigureVSCode has to download
  every reinstall, which is what we're trying to avoid.
- (B) Add `--keep-codex` flag to agentctl uninstall, default off; install.ps1
  passes `--keep-codex` so a re-install can reuse the cached binary.
- (C) (Best) Keep the download in `%LOCALAPPDATA%\agentserver-vscode\bin\`
  and only delete it when user explicitly opts in (`agentctl uninstall --codex`).

### Bug #3: agentctl doctor doesn't show state when state.json missing

Repro: Fresh install, no `state.json` yet. `agentctl doctor` shows fresh
`install_id` each call. Confirmed: `state.Load()` returns `freshState()`
in-memory but doesn't save, so two `doctor` calls show different IDs.

Not really a bug — it's just confusing. Maybe doctor should print
"(state.json not yet written)" when stat fails on the file.
