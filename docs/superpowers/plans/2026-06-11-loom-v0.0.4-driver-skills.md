# Loom v0.0.4 Driver Skills Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade packaged Loom assets to `v0.0.4` and install Loom driver skills/prompts for Codex Desktop and minimal VS Code Codex users.

**Architecture:** Packaging scripts pin and cache the v0.0.4 executable and support archives. A focused `internal/loom` helper unpacks managed skills/prompts safely. Existing onboarding and completed-launch flows call the helper before registering the driver MCP server in Codex config.

**Tech Stack:** Go, PowerShell, Bash packaging scripts, Inno Setup, TOML/YAML config writers.

---

## File Structure

- `internal/codex/config.go`: extend `MCPServer` to support timeout and enabled fields in TOML.
- `internal/codex/config_test.go`: lock MCP server TOML output.
- `internal/loom/support.go`: install Loom driver skills and Codex prompt assets from release tarballs.
- `internal/loom/support_test.go`: cover safe extraction and managed prompt merge.
- `internal/loom/config.go`: add support archive paths to `DriverConfig` or a sibling runtime input only if needed.
- `internal/ui/orchestrator_real.go`: install driver support during onboarding frontend configuration.
- `internal/ui/orchestrator_real_test.go`: assert first-run Codex Desktop configuration installs support files.
- `cmd/launcher/main.go`: install driver support on completed launches.
- `cmd/launcher/main_test.go`: assert completed launch installs support files.
- `scripts/package-windows.sh`: bump Loom to v0.0.4 and download support archives.
- `scripts/package-windows-zip.sh`: same for portable package and copy archives into the stage directory.
- `packaging/windows/installer.iss`: include v0.0.4 assets and support archives.
- `packaging/windows/install.ps1`: require and copy support archives for portable installs.
- `internal/vscode/install_test.go`: update Windows packaging assertions.

The first test creates tar.gz fixtures with `skills/multiagent/SKILL.md` and `prompts-codex/AGENTS.md`, calls `InstallDriverSupport`, and asserts files exist under `.agents/skills/multiagent`, `.codex/skills/multiagent`, and `.codex/AGENTS.md`.

- [ ] **Step 3: Implement minimal support installer**

Create:

```go
type DriverSupportInput struct {
	UserHome              string
	SkillsArchivePath     string
	CodexPromptsArchivePath string
}

func InstallDriverSupport(in DriverSupportInput) error
```

Behavior:

- Return nil if `UserHome` is empty.
- If `SkillsArchivePath` exists, extract `skills/*` into `UserHome/.agents/skills` and `UserHome/.codex/skills`.
- If `CodexPromptsArchivePath` exists, extract `prompts-codex/AGENTS.md` to a temp dir and merge it into `UserHome/.codex/AGENTS.md`.
- Reject absolute archive names and `..` path escapes.
- Preserve content outside markers:

```text
<!-- agentserver-app loom driver prompt:start -->
...
<!-- agentserver-app loom driver prompt:end -->
```
